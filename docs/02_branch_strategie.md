# Branch-Strategie

## main
Der Branch `main` enthält immer den aktuellen stabilen Stand des Projekts.

## Feature-Branches
Neue Aufgaben und Änderungen werden in eigenen Branches wie `feat/Hauptseite` oder `feat/Ignoranz` umgesetzt.

## Pull Requests
Änderungen werden nicht direkt in `main` gemacht, sondern zuerst per Pull Request geprüft und danach gemerged.

## Ziel
Diese Strategie sorgt für mehr Ordnung, bessere Nachvollziehbarkeit und einen stabilen Hauptbranch.
